# Blinkit Product & Inventory Analysis using PostgreSQL

## Project Overview

This project analyzes Blinkit's product and inventory dataset using PostgreSQL.

The objective is to extract business insights related to product profitability, inventory planning, pricing strategy, brand performance, and shelf-life management.

The project simulates real-world Data Analyst responsibilities by answering business questions through SQL queries.

---

## Tools Used

- PostgreSQL
- SQL

---

## Dataset Information

The dataset contains product-level information including:

- Product Name
- Category
- Brand
- Price
- MRP
- Margin Percentage
- Shelf Life
- Minimum Stock Level
- Maximum Stock Level

---

## Business Problems Solved

### Product Analysis

- Total Products Count
- Total Categories Count
- Total Brands Count
- Minimum and Maximum MRP
- Top 10 Products by MRP

### Profitability Analysis

- Top Categories by Margin Percentage
- Top Brands by Margin Percentage
- Most Profitable Category
- Most Profitable Brand

### Inventory Analysis

- Inventory Buffer Capacity Analysis
- Stock Planning Recommendations
- Category-wise Stock Buffer Evaluation

### Shelf-Life Analysis

- Average Shelf Life by Category
- High Perishability Categories
- Shelf-Life Risk Identification

---

## Key SQL Concepts Used

- Aggregate Functions
- GROUP BY
- ORDER BY
- HAVING
- Subqueries
- DISTINCT
- Business KPI Analysis

---

## Key Insights

- Identified the most profitable product categories.
- Identified high-margin brands for marketing focus.
- Evaluated inventory buffer capacity for stock planning.
- Identified categories with high perishability risk.
- Generated actionable business recommendations using SQL analysis.

---

## Business Recommendations

1. Increase inventory allocation for high-profit categories.
2. Focus marketing efforts on high-margin brands.
3. Closely monitor categories with low shelf life to reduce inventory losses.
